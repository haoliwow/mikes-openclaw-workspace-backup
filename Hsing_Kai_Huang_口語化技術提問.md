# Hsing Kai Huang — 4 個口語化技術提問（顧問角度）

**角色設定**：獵頭顧問，了解他的技術背景和工作方式，評估是否適合新機會

---

## ❓ 提問 1：消息隊列 & 非同步處理

### 對標 Java #52 需求
JD 提到「OpenAPI/Message Queue 使用經驗」

### 口語化提問

```
「看你的履歷，在 Athena 做了不少架構設計的工作。
我比較好奇的是——在那邊有沒有接觸過消息隊列這類東西？

比如說 Kafka、RabbitMQ 這種，或者其他的異步處理方案？
不一定要很深，只是想了解你們在系統裡是怎麼處理高併發的場景。」
```

### 為什麼這樣問
- ✅ 開放式，讓他自由說故事
- ✅ 「不一定要很深」降低壓力感
- ✅ 問的是「實際遇過什麼」而不是「會不會」
- ✅ 為後續的「系統架構思想」鋪路

### 你要聽什麼
- 他有沒有接觸過 MQ（有最好，沒有也不致命）
- 如果有，是在什麼場景下用的（高併發？非同步任務？）
- 他對「為什麼用 MQ」的理解有多深
- 如果沒有，他對「學習新技術」的態度怎麼樣

## ❓ 提問 2：自動化部署 & 開發流程

### 對標 Java #52 需求
JD 提到「配合 CI/CD、DevOps 等開發模式」

### 口語化提問

```
「你在 Athena 推動了 Agile 和一些新的開發流程。
我想了解一下——你們的 code 從寫完到上線，整個流程是怎樣的？

有沒有自動化的東西？比如說，有沒有用過什麼工具來自動化測試或部署？」
```

### 為什麼這樣問
- ✅ 不直接問「會不會 GitLab CI」（太硬）
- ✅ 問的是「他在的公司怎麼做」（更自然）
- ✅ 暗示他可能是「推動流程改革的人」
- ✅ 為評估「適應變化能力」做準備

### 你要聽什麼
- 他們現在的 CI/CD 成熟度
- 他有沒有參與過流程改進
- 他對「自動化」的重視程度
- 他學習新工具的速度（間接評估適應性）

---

## ❓ 提問 3：微服務通訊 & 系統整合

### 對標 Java #52 需求
JD 提到「微服務經驗（維護或開發都可）」

### 口語化提問

```
「你提到了資料庫層的整合設計，做得蠻深的。
我想進一步了解——除了資料庫這一塊，
你們的系統裡有沒有拆分成多個微服務？

如果有的話，這些服務之間怎麼溝通？
是直接 call API，還是透過什麼中介？」
```

### 為什麼這樣問
- ✅ 承接上一個提問（MQ）但更深一層
- ✅ 問的是「架構設計的思路」而不是「技術細節」
- ✅ 幫助評估他對「系統整體設計」的理解
- ✅ 自然地區分「他做過什麼」vs「他理解多深」

### 你要聽什麼
- 他們的系統是單體還是已微服務化
- 如果微服務了，他在其中扮演什麼角色
- 他怎麼看待服務間通訊的 trade-off（速度 vs 複雜度）
- 他有沒有參與過「系統拆分決策」

---

## ❓ 提問 4：系統設計的決策邏輯

### 對標 Java #52 需求
JD 提到「能快速學習且適應變化」+ 「系統分析實務經驗」

### 口語化提問

```
「你從 Caresys 的『30x 效能提升』，進階到 Athena 的『系統整合設計』。
這個轉變很有意思——代表你的思維從『怎麼優化單點』升級到『怎麼設計整個系統』。

能聊一下嗎？當你決定『用 Keycloak 而不是自己寫 SSO』的時候，
你怎麼評估這個選擇的？有沒有考慮過其他方案？」
```

### 為什麼這樣問
- ✅ 肯定他的進階（建立好感）
- ✅ 問的是「決策邏輯」而不是「技術細節」
- ✅ 評估他是否真的在「思考」而不只是「執行」
- ✅ 看出他對「trade-off」的理解（架構師級思維）

### 你要聽什麼
- 他是怎麼評估「買 vs 自己寫」的
- 他考慮過哪些 Keycloak 的替代方案（如果有的話）
- 決策時誰參與了（他主導？還是被動接受？）
- 他對「系統設計中的權衡」有多深的理解

## 📍 4 個提問的對話流

**別按順序問。根據他的回答自然延伸：**

```
開場：
「感謝你的時間。我看了你的履歷，很印象深刻。
今天主要想從技術背景這個角度聊一下。」

↓

從 Athena 的工作切入（最自然）：
「你在 Athena 這 2.5 年做了不少不一樣的事...」

↓

根據他的回答延伸：

如果他說「做了很多系統設計」
  → 問提問 3（微服務通訊）和提問 4（決策邏輯）

如果他說「推動了 Agile 和新流程」
  → 問提問 2（CI/CD 工作流程）

如果他提到「複雜的工程」
  → 問提問 1（MQ 經驗）

↓

最後一個提問（提問 4）放在對話的尾聲：
「再問一個——當你決定『用什麼技術』的時候...」
```

## 🎯 核心口語技巧

| 不要這樣 | 應該這樣 |
|---------|--------|
| 「你有 Kafka 經驗嗎？」| 「在 Athena 有沒有接觸過 MQ？」 |
| 「你會用 GitLab CI 嗎？」| 「code 到上線的流程怎樣？有自動化嗎？」|
| 「你了解微服務嗎？」| 「你們的系統有拆分成多個服務嗎？」|
| 「系統設計的最佳實踐是什麼？」| 「當你選擇 Keycloak 時，怎麼評估的？」|

**重點：問的是「他的故事」而不是「他的知識」**
