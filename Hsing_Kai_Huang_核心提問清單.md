# Hsing Kai Huang — 核心提問清單

**目的：了解「這個機會是否適合他」，而不是「推銷機會」**

---

## 🎯 5 個核心提問

### **#1：職涯選擇的邏輯**
```
「我看你從 Caresys 轉到 Athena。
當時怎麼選擇的？什麼吸引你從安全監控進到金融科技？」

你要聽的：
├─ 他選工作時，最在意什麼？(技術？薪資？公司規模？業務複雜度？)
├─ 他是被動接受機會，還是主動尋求？
└─ 他對「產業」vs「技術」的看法

根據回答判斷：
• 如果他說「想要更複雜的系統」→ 他在乎架構、系統設計
• 如果他說「薪資更好」→ 他可能對工作內容不是最關心
• 如果他說「想學新東西」→ 他是學習驅動的
```

---

### **#2：現在的不滿足點**
```
「現在考慮轉職，有什麼具體的想法嗎？
Athena 2.5 年，是達成了什麼目標，還是...？」

你要聽的：
├─ 他在 Athena 達成了什麼（有成就感？有瓶頸？）
├─ 為什麼現在要走（天花板？想換環境？個人計畫？）
└─ 他對「下一步」有沒有清晰的想法

根據回答判斷：
• 如果他說「想要更多決策權」→ 他在大公司可能被層級限制了
• 如果他說「想嘗試新領域」→ 他需要新鮮感、新挑戰
• 如果他說「公司文化不適」→ 他在意團隊氛圍和決策速度
• 如果他說「沒具體想法，但該轉了」→ 他是被動的，可能容易反悔
```

---

### **#3：技術方向的真實興趣**
```
「你在 Caresys 重點是性能優化，Athena 反而做更多架構設計。
這個轉變是你有意的，還是職位本身就這樣？」

你要聽的：
├─ 他對「性能優化」vs「系統架構」哪個更有熱情
├─ 他是被職位推著走，還是主動選擇這個方向
└─ 他對「下一步」想深耕哪個領域

根據回答判斷：
• 如果他說「我發現自己更喜歡架構」→ 他的技術方向很清晰，想往這走
• 如果他說「職位就是這樣」→ 他可能沒有很強的技術偏好，比較實用主義
• 如果他說「兩個都喜歡，想都做」→ 他想要 full-stack 的機會
```

---

### **#4：推動變革時的體驗**
```
「我看你推動了 AI×BDD 和 Agile 這些改革。
在 Athena 推動新東西時，最大的挑戰是什麼？」

你要聽的：
├─ 他有沒有遇過「決策慢」「層級多」「買不動」的問題
├─ 他在推動技術變革時的「挫折感」有多大
└─ 他對「快速決策」的渴望程度

根據回答判斷：
• 如果他說「決策層級太多」→ 他渴望更小、更靈活的環境
• 如果他說「團隊配合度不高」→ 他在意能夠快速執行想法
• 如果他說「沒有挑戰，很順利」→ 他在 Athena 可能還滿足，轉職原因不是這個
```

---

### **#5：職業發展的真實期望**
```
「展望接下來 1-2 年，你希望自己在技術上達成什麼？
或者說，理想中的『下一個角色』是什麼樣的？」

你要聽的：
├─ 他想往「架構師」走，還是「技術主管」，還是「深耕某技術領域」
├─ 他對「影響力」的定義（寫好代碼？定義系統？帶人？）
└─ 他在意「職級/薪資」還是「做什麼事」

根據回答判斷：
• 如果他說「想成為架構師」→ Java #52 能給他「定義系統」的機會
• 如果他說「想帶人/做管理」→ Java #52 可能不太適合（除非有團隊）
• 如果他說「想嘗試新技術/新領域」→ 要看 Java #52 能不能提供
• 如果他說「不知道，但想成長」→ 他的目標不清晰，可能需要指導
```

## 🔄 對話流程

**不要按順序問，要根據他的回答自然延伸：**

```
開場：
「感謝你的時間。我看了你的履歷，蠻印象深刻。
想聊一下你的職涯想法，看看我們的機會是不是適合。」

↓

從 #1 開始（職涯選擇）：
「我看你從 Caresys 轉到 Athena...」

↓

根據他的回答，往 #2 or #3 延伸：

如果他強調「想要更複雜的系統」
  → 往 #3 問「那在架構設計上有興趣嗎」

如果他提到「大公司的限制」
  → 往 #4 問「推動變革時遇過什麼困難」

如果他比較被動
  → 直接往 #5 問「那你期望的下一步是什麼」

↓

最後（他說完了，你再說）：
「你說的這些，其實和我們這個機會...」
（根據他的回答，再評估機會是否 match）
```

---

## 📊 聽完後的判斷框架

提問完 5 題後，你應該能知道：

| 他想要的 | 對應的信號 | Java #52 能不能給 |
|---------|---------|---------|
| **架構決策權** | 強調「想定義系統」、「推變革困難」 | ✅ 能，這正是職位的核心 |
| **決策速度快** | 強調「大公司層級多」、「執行慢」 | ✅ 能，小公司決策更快 |
| **新領域挑戰** | 強調「想離開金融」、「想試新東西」 | ? 要看 Java #52 的業務領域 |
| **技術深度** | 強調「想在 XX 技術深耕」 | ? 要看技術棧是否符合 |
| **帶人/管理** | 強調「想帶團隊」、「想做 leader」 | ❌ 可能不能，除非有升遷空間 |
| **薪資優先** | 一直問薪資、福利 | ? 要看預算是否符合 |

---

## 💡 重點

**不要一開始就說「我們的機會有 8 個優勢」**

**要先問 5 個問題，了解他：**
- 他的選擇邏輯是什麼
- 他現在卡在哪
- 他真正想要什麼
- 他對「下一步」的想法

**然後才說：**
「根據你說的這些，我覺得 Java #52 對你可能有這些價值...」

這樣才不會空洞，才是真的「這個機會是不是適合他」，而不是「我來推銷」。

---

## ✅ 這樣的提問，是你要的嗎？

這 5 個提問能幫你：
1. ✅ 建立信任感（聽他說，而不是你說）
2. ✅ 了解他真實的職涯想法
3. ✅ 發現「Java #52 對他的特定價值」在哪
4. ✅ 判斷「這個機會是不是真的適合他」
5. ✅ 如果適合，知道用什麼角度來說服他
