
### 職缺群客戶名單匯入流程（Phoebe 指定 - 2026-04-07）

- 當 Phoebe 說「匯入職缺群名單」「匯入職缺群sheet」「匯入職缺群客戶名單」，並提供 IG/Threads/對話截圖時：
  - 任務：
    - 從截圖中抽取：
      - IG/Threads 帳號 → C 欄
      - Line 名稱 / Line id → D 欄
      - 目前產業 → E 欄
      - 職稱（現職或目標職，若未提供可註明「未提供」）→ F 欄
      - 到職/待業狀態（就職中/就業/待業/全職創業等）→ G 欄
      - 想看的職缺類型（整理成一句話）→ H 欄
    - A 欄為「編號」，從 Phoebe 指定的起始號碼往下遞增（例如這次從 110、後來 114、117、119、121...）。
    - B 欄為日期，使用當天日期（例如 2026/04/07）。
    - I 欄「履歷提供」預設留空，之後有履歷再標。
  - 目標表格：
    - Google Sheet：`職缺群客戶名單`（`履歷池 v2` → 工作表1）
    - Sheet ID：`1XpH0jwmbLdVxMz4uud6bYt-NXKll91tc1qOVuk0HkDo`
    - 工作表名稱：`工作表1`
  - 技術實作：
    - 不再使用 `gog sheets append`（容易整串塞一格）。
    - 一律用 `gog sheets update`，逐格寫入：
      - 範例：
        - `gog sheets update "<sheetId>" "工作表1!A108" "110" --account "haoliwow@gmail.com"`
        - 依序更新 A~I 欄位。
    - 遇到 vision 把 IG 截圖誤認成別人（如 goldencat1111/verina0105 混淆）時：
      - 改用 Phoebe 提供或描述的「文字版資訊」為準，不再信任當次圖片解析結果。
