# 📋 日報告模板 & 流程 (Daily Report SOP)

## 發布時間
- **初稿提交**：每日 19:00
- **最終發布**：每日 20:00

## 報告流程
1. **早上開會**（Phoebe 與 Mike）→ 確認今日計畫與優先順序
2. **全天執行**（Mike） → 記錄進度、數字、實際完成項目
3. **下午 19:00** → Mike 提交初稿（含所有實際數字）
4. **Phoebe 審核並補充** → 告訴我哪些做好了、填入數字
5. **下午 20:00** → 發布最終版本

## 關鍵原則

✅ **DO**
- 只填寫**實際完成的數字**（不估計、不假設）
- 有完成就填，沒完成就空著，**等 Phoebe 說**
- 19:00 準時提交初稿
- 每天早上開會記錄當日計畫到記憶

❌ **DON'T**
- 不要編造數字
- 不要假設 Phoebe 說了什麼（要問）
- 不要遲交初稿
- 不要內容沒做到卻寫進去

## 記憶更新規則
- 每日早上開會內容 → 寫入 `memory/YYYY-MM-DD.md`
- 全天進度 → 實時記錄
- 19:00 初稿 → 交給 Phoebe 審核
- 20:00 發布版 → 最終確認
