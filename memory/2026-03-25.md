## 年資與薪資匹配警告（2026-03-25 新規則）

**重要觀察**：當候選人年資 >> 職缺需求年資時，需特別提醒薪資期待風險

**林建均案例**：
- 職缺 #224（PG-BE 後端工程師）需求：Junior-Mid（薪資 45-70k）
- 候選人：7.42 年資，現為 Mid-Senior 等級
- **風險**：候選人期望 85-110k，職位薪資 45-70k → **薪資期待不符 (落差 15-65k)**

**正確做法**：
在 Job Fit 分析與 phone_scripts 中，明確提及：
- ❌ 「職級可能不符」
- ❌ 「薪資可能低於期望」
- ✅ 「職位薪資 X k，候選人期望 Y k，差距明顯」
- ✅ 在 phone_scripts 第一題就確認薪資期望
- ✅ 在 recommendation 提醒「薪資談判可能困難」

**應用於後續候選人**：
每次看到「年資過剩」的配對（年資是職缺需求的 2 倍以上），都要：
1. 在 main_gap 明確標註薪資期待風險
2. 在 phone_scripts 中 is_veto=true 的薪資確認題中提到「職位薪資範圍」
3. 在 recommendation 中評估「薪資談判難度」

### 三步驟完整流程：

**Step 1：POST /api/candidates (建檔)**
- **必填欄位**：name, current_position, total_years, skills, location, source, status, recruiter, linkedin_url
- **強烈推薦**：education_details（見下方格式）、work_history

**Step 2：POST /api/candidates/{id}/resume (上傳履歷)**
- 上傳 PDF 檔案
- 取得 file_id（會用在 Step 3）

**Step 3：PUT /api/ai-agent/candidates/{id}/ai-analysis (AI 分析)**
- 完整 AI 分析格式（見格式規範文件）
- 包含：candidate_evaluation + job_matchings + phone_scripts + recommendation

### education_details 格式（關鍵）

**建檔時 (import-complete 或 POST /api/candidates)：**
- 欄位名：`education_details`（snake_case）
- 格式：陣列
- **必填** ← 第一次匯入就要帶，不能事後補

```json
"education_details": [
  {
    "school": "學校名稱",
    "degree": "碩士/學士/結業",
    "major": "科系",
    "graduation": "2023"
  },
  {
    "school": "學校名稱",
    "degree": "學士",
    "major": "科系",
    "graduation": "2019"
  }
]
```

**補寫時 (PATCH /api/candidates/{id})：**
- 欄位名：`education_details`（snake_case，NOT camelCase）
- 格式：同上陣列格式

**常見錯誤**：
- ❌ `educationDetails`（camelCase）→ API 拒絕
- ❌ `education`（欄位不存在）→ 無效
- ❌ 物件格式而非陣列 → API 拒絕
- ✅ `education_details` + 陣列格式 → 正確
