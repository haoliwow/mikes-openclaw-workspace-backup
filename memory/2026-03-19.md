## 核心學習：正確的候選人匹配流程

### ❌ 我的錯誤模式
1. 只看 skills 欄位裡的關鍵字（e.g., "Airflow"、"ETL"）
2. 就直接說「符合」推薦出去
3. **根本沒驗證實際工作經歷描述**

### ✅ 正確流程（Phoebe 教的）
1. **先拿完整工作經歷描述**
2. **逐項檢查職缺需求**（不是估計）
3. **全部符合才推薦**（說明缺什麼）
4. **只有無法確認的才說「需要補充工作經歷」**

### 回答方式改進
舊方式：「#1086 符合 ETL/Pipeline」
新方式：「#1086 有 Pentaho/Airflow，但缺乏 Kafka/Spark，無法準確評估。建議：需要補充工作經歷才能準確匹配。」

## 重要認知轉變

**Phoebe的教導：**
> 「你可以改告訴我：這個可能相近但需要補充工作經歷才能準確。這樣的回答」

**含義：**
- 承認不確定性 > 假裝確定
- 說明缺什麼 > 隱瞞缺陷
- 需要補充 > 不符合

**應用到開發信：**
- 只推送有具體工作經歷驗證的人選
- 若只有skills標籤，要說明「可能相近，需電話確認」

   - [ ] 逐項檢查職缺5-8個核心需求
   - [ ] 明確說出「符合」「需要補充」「不符合」

2. **開發信前的驗證步驟：**
   - [ ] 確認最少3項硬性需求都符合
   - [ ] 缺什麼也要寫在notes裡

3. **溝通改進：**
   - 不隱瞞不確定性
   - 用「可能相近但需補充工作經歷」替代「符合」
