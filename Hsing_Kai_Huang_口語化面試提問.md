# Hsing Kai Huang — 口語化技術對話指南

**面試 Tone：同行對話，而不是考問**  
**目標：了解技術背景 + 建立信任 + 判斷 match 度**

---

## 📝 對話開場

```
「昆凱，我看了你的履歷，印象很深。你從 Caresys 到 Athena，
技術棧的演進看得出來。想利用這個機會多了解一下你的背景。
我們的許多決策都會參考你們過往做過的方案，所以蠻想聽聽你的故事。」
```

---

## 🎯 對話流 (8 大問題 — 口語化版)

### **Part 1：產品與架構背景（建立共識）**

#### ① 產品形態 & 微服務狀況
```
「看完你的履歷想了解一下，你在 Athena 和 Caresys 負責的產品，
主要偏什麼方向的？是金融類的？

另外，我比較好奇的是——你們的系統架構是怎樣的？
有沒有涉及微服務？還是說相對傳統的單體應用？
如果有，大概拆成幾個服務，怎麼做的？」

設定期望：
- 自然地問「是什麼」（不是考「懂不懂」）
- 讓他自己講故事
- 根據回答再延伸
```

---

### **Part 2：技術深度（同行視角）**

#### ② 資料庫抽象層的設計思路
```
「你提到做了『資料庫抽象層，同時支援 Oracle 和 PostgreSQL』。
我覺得這個蠻酷的。能聊聊你們怎麼處理的嗎？

比如說，你們用的是 Hibernate/JPA？還是自己寫的 ORM？
遇過什麼棘手的兼容性問題嗎？比如序列啦、資料型別啦這些。」

設定期望：
- 用「我覺得這很有趣」來表達認可
- 問的是「怎麼做的」而不是「為什麼這樣做」
- 給例子，讓他知道我們在這個領域有經驗
```

#### ③ 那個 30x 性能提升的細節
```
「我特別想聽一下——你提到 database ingestion 做到 30x 提升。
這是怎麼達成的？

是純粹的並發優化？還是有涉及其他的？
比如說連線池調整、批量提交、還是什麼？
那時候的瓶頸是在 CPU、IO 還是資料庫？」

設定期望：
- 問的是「你怎麼思考的」，不是「你對不對」
- 讓他展示解決問題的過程，而不只是結果
- 聽他如何做效能分析、決策、驗證
```

#### ④ Keycloak 零停機遷移怎麼規劃的
```
「SSO 這種核心基礎設施遷移，我能想像風險很大。
你們怎麼確保零停機的？是灰度發布？還是有什麼其他策略？

遇過什麼惱人的問題嗎？比如 session 同步啦，或者 fallback 方案。」

設定期望：
- 顯示我們理解這類遷移的複雜性
- 問的是「過程」和「思考」，而不是「結果」
- 帶著「我也想學習」的態度
```

---

### **Part 3：故障處理 & 系統觀察（判斷工程文化）**

#### ⑤ Arthas 診斷的實例
```
「你提到用 Arthas 做實時診斷和 hotfix。
我蠻好奇的是——具體在什麼情況下用的？

比如遇過記憶體洩漏？CPU 飆高？還是什麼異常行為？
當時是怎麼發現問題、怎麼用 Arthas 定位、然後怎麼修的？」

設定期望：
- 聽他的故障處理方法論
- 判斷他有沒有真的用過，不只是聽說
- 看他對 JVM 的理解深度
```

---

### **Part 4：職業與期望（軟技能 & 文化契合）**

#### ⑥ 為什麼考慮下一步？
```
「我看你在 Athena 當 Senior Engineer，技術環境看起來也不錯。
現在考慮轉職，主要是什麼原因？

你對下一個環境的期望是什麼？是想要更多的架構責任？
還是說想在某個技術方向上深耕？或者是公司文化、團隊規模的考量？」

設定期望：
- 聽他的成長動機，而不是假設
- 判斷他的穩定性
- 了解他重視什麼
```

#### ⑦ 怎麼推動技術變革的？
```
「我看你的履歷裡提到『Championed Agile』和『Pioneered AI×BDD』。
能分享一個具體的例子嗎？

比如說——你怎麼說服整個團隊採用新的工具或流程的？
有沒有遇過反對？當時是怎麼處理的？」

設定期望：
- 了解他的領導力和影響力
- 聽他怎麼處理技術決策的 friction
- 判斷他在團隊中的角色和風格
```

---

### **Part 5：雙向確認（他的問題 & 我們的期望）**

#### ⑧ 你對我們的團隊、職位有什麼問題嗎？
```
「以上就是我們想了解的。你對我們的角色、公司、或工作內容
有什麼疑問嗎？

我們也想確認一下——
這個職位的期望是什麼樣的技術棧和工作內容？
有沒有什麼是特別吸引你的，或者是需要進一步確認的？」

設定期望：
- 給他提問的空間（好的候選人會有準備好的問題）
- 根據他的問題，判斷他的關注點是什麼
- 為接下來的薪資談判、offer 做準備
```

## 🎬 對話節奏與技巧

### **開場 (5-10 分鐘)**
```
「感謝你的時間。我看了你的履歷，印象最深的是 ___。
今天主要想從三個角度聊一下：
1. 你過往的技術背景和產品
2. 一些技術決策的思路
3. 了解一下你對下一步的期望

都是開放式的對話，沒有標準答案。」
```

### **技術深度 (20-30 分鐘)**
- ✅ 用「我很好奇」開頭
- ✅ 一個問題講完，停下來聽
- ✅ 根據他的回答自然延伸（不一定按順序問）
- ✅ 他講得好就多聽，深入追問；講不清楚的就換個角度
- ✅ 不要打斷，讓他完整表達

### **軟技能 & 期望 (10-15 分鐘)**
- ✅ 轉換到「工作風格」和「期望」
- ✅ 不要突然很正式，要自然過渡
- ✅ 聽他對工程文化的理解

### **反向提問 (10 分鐘)**
- ✅ 給他充分的提問時間
- ✅ 他問什麼，就回答什麼（不要隱瞞）
- ✅ 記下他問的問題，了解他的優先順序

## 💡 回答技巧（如果他問我們）

### 如果他問：「你們的系統架構是什麼？」
```
「我們現在用的是微服務架構，Java/Spring Cloud 體系。
但坦白說，我們在高科技廠房的 BIM 標準化方面還在探索。
這也正是為什麼我們需要你這樣的 Senior Engineer——
定義什麼是好的、可複製的架構。」
```

### 如果他問：「這個職位的具體工作內容是什麼？」
```
「核心是——我們在做微服務系統，需要有人能從架構層面
思考『怎麼設計系統才能長期可維護、可擴展』。

不是單純的『寫代碼交付』，而是『參與系統設計決策』。
Year 1 你會接觸現有系統的 refactor，同時定義新模組的架構方向。」
```

### 如果他問：「薪資和福利怎樣？」
```
「薪資部分，我們的範圍是 60-80k，上限有談的空間，
特別是如果技術深度符合期望的話。

福利方面——我現在給不了確切的數字，但 HR 後續會詳細說明。
我們會在 offer 階段完整討論。」
```

## 🎯 評估重點（邊聽邊評）

聽他回答時，註意：

| 項目 | 觀察 |
|------|------|
| **清晰度** | 能不能清楚解釋複雜的技術決策 |
| **深度** | 是不是只知道表面，還是真的理解 |
| **實戰感** | 講的是真實的故事，還是紙上談兵 |
| **系統思考** | 能不能從整體考慮，不只是單點優化 |
| **團隊協作** | 怎麼和別人互動、說服、協調 |
| **學習心態** | 願不願意學新東西、面對挑戰 |
| **匹配度** | 他的技術方向和期望，和我們的職位吻合嗎 |

## 📋 面試後檢查清單

**面試結束前確認：**
- [ ] 確認他對職位有清楚的理解
- [ ] 了解他的期望薪資範圍
- [ ] 了解他的 availability（什麼時候能開始）
- [ ] 講清楚下一步的流程和時間表

**面試結束後記錄：**
- [ ] 技術深度評分（1-10）
- [ ] 文化契合度（1-10）
- [ ] 他特別關心的是什麼
- [ ] 有沒有紅旗（stability issues, technical gaps 等）
- [ ] 推薦度（Strong Yes / Yes / Maybe / No）

---

## 🎁 Tips 讓他感受到信任度

1. **認可他的履歷亮點**  
   「30x 效能提升這事，很多人說說而已，你是真的做到」

2. **展示我們的深度**  
   「我們現在的挑戰是 ___，你在 Athena 的經驗對我們很有啟發」

3. **坦誠分享我們的限制**  
   「坦白說，我們的團隊規模比不上大公司，但決策快」

4. **聽的時候用眼神接觸、點頭**  
   Show 出你在認真聽，不是在表演

5. **他講完了，停頓 2-3 秒再說話**  
   給他補充的空間
