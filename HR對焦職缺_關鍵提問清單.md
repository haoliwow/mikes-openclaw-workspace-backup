# HR 對焦職缺 - 關鍵提問清單

**建立日期**: 2026-03-17  
**用途**: 每次新增或更新職缺時，與 HR/客戶對焦時提出的額外問題  
**負責人**: Jacky / Phoebe

---

## 📋 必問的三大問題

### 1️⃣ 遠端工作機會

**問題**: 「這個職位有遠端工作或彈性工作的機會嗎？」

**為什麼重要:**
- 遠端/彈性工作是吸引優秀人才的重要因素
- 許多候選人（特別是資深工程師）會優先考慮彈性工作安排
- 可能擴大可推薦的候選人池（跨地域）

**追問細節:**
- [ ] 完全遠端？還是混合工作（幾天在辦公室）？
- [ ] 是否接受跨城市遠端（e.g., 台北人在台中工作）？
- [ ] 試用期內是否有遠端限制？
- [ ] 核心會議時間是否要在辦公室？

**記錄方式**: 在職缺系統的「remote_work」欄位清楚標註

---

### 2️⃣ AI 相關導入資訊

**問題**: 「目前公司有 AI 相關導入的資訊嗎？未來有 AI 應用計畫嗎？」

**為什麼重要:**
- AI 是當下人才市場最熱門的方向
- 有 AI 轉型計畫的公司更容易吸引優秀人才
- 幫助候選人了解職位的成長潛力

**追問細節:**
- [ ] 公司是否有導入 AI 工具（e.g., ChatGPT, Copilot 等）？
- [ ] 技術團隊是否有 GenAI / 機器學習相關專案？
- [ ] 是否計畫在該職位涉及的團隊導入 AI？
- [ ] 公司在 AI 轉型上的投資規模？
- [ ] 員工 AI 培訓計畫是否存在？

**記錄方式**: 在職缺備註欄新增「AI 導入狀況」小段落

**例子:**
```
【AI 相關資訊】
- 公司已導入 ChatGPT Enterprise
- 資料團隊正在開發 GenAI 應用（RAG 系統）
- 技術團隊每季有 AI 工具探索會議
```

---

### 3️⃣ 人選品質下降時的容忍度

**問題**: 「如果我推薦的人選很不錯，但經驗稍微 junior 一點（e.g., 少 1-2 年），你們會考慮嗎？」

**為什麼重要:**
- 了解職缺的「硬門檻」vs「軟門檻」
- 幫助精準篩選候選人（不浪費雙方時間）
- 發現高潛力但經驗略低的人才時，有談判空間

**追問細節:**
- [ ] 年資要求是「必須」還是「最好有」？
- [ ] 如果人選技能 90+ 分，經驗少 1-2 年，能接受嗎？
- [ ] 是否願意考慮「有潛力但需要 3-6 個月 ramp-up」的人選？
- [ ] 如果不能接受 junior 人選，原因是什麼？（時間壓力？項目複雜度？）
- [ ] 有沒有 mentor/onboarding 資源支持新人成長？

**記錄方式**: 在職缺系統的「talent_profile」或「consultant_notes」欄位記錄

**例子:**
```
【人選彈性度】
- 年資要求：「最好 5yr+，但如果人選技能強、有意願學習，4yr 也能考慮」
- 優先考慮：「有大型專案經驗 > 年資多」
- Ramp-up 支持：「有資深師傅帶，預計 3 個月能獨立」
```

## 📝 額外補充問題（視狀況提出）

### 4️⃣ 薪資彈性空間

**問題**: 「職缺寫面議或給定範圍，實際有多少談判空間？」

**記錄**: 在薪資欄位備註真實預算範圍

**例子**: 
```
開價: 面議
實際預算: 140-160M（若技能特別強可到 170M）
```

---

### 5️⃣ 緊急程度與時間壓力

**問題**: 「這個職位有招募截止日期嗎？還是長期開放？」

**記錄**: 在職缺備註欄標註

**例子**:
```
【招募時間表】
- 優先招募截止: 2026-04-30
- 預計上班日: 2026-05-15
- 若找不到理想人選，會考慮備選條件
```

---

### 6️⃣ 轉職成本與風險

**問題**: 「候選人轉職需要考慮的限制因素是什麼？（e.g., 地點、工時、出差、技術棧轉換難度）」

**記錄**: 在職缺備註欄標註「轉職風險」

**例子**:
```
【轉職風險因素】
- 地點: 新北五股（需從台北通勤）
- 出差: 需要不定時國內出差
- 技術: 需從 Java 轉到 C#（學習曲線陡）
- 改善方案: 提供遠端工作日、出差補助、技術培訓
```

## 🎯 使用流程

### 何時使用

**新職缺加入時:**
- [ ] 初次與 HR/客戶對焦
- [ ] 填寫職缺系統前

**職缺更新時:**
- [ ] 每月 1-2 次定期對焦
- [ ] 如果一直沒找到合適人選，重新評估

---

### 怎麼使用

**形式 1: 對焦會議（最佳）**
```
Jacky 與 HR 線上/面對面開會
- 時間: 15-30 分鐘
- 準備: 帶著這份清單
- 記錄: 在職缺系統備註欄填充答案
```

**形式 2: 郵件提問**
```
Subject: [職缺對焦] #216 Playsee PM - 額外確認 3 點

Hi [HR Name],

關於 #216 Playsee PM 職位，我在推薦前想確認三個重點:

1. 這個職位有遠端工作的機會嗎?
2. 公司目前有 AI 相關導入或轉型計畫嗎?
3. 如果人選技能很強但經驗少 1-2 年，願意考慮嗎?

Thanks!
Jacky
```
