# 客戶開發信 - Tier 分層模板（Jacky 格式）

**目標**: 為不同客戶階層準備即用型開發信  
**格式**: 向雇主推薦候選人（不是向候選人推銷職位）  
**發送渠道**: LinkedIn DM / Email

---

## 📍 Tier 1：新創 (Series A/B) 開發信

### 特徵
- 決策快、缺人急、有預算
- CEO/CTO 通常親自管招聘
- 月薪 5-8 萬起
- 預期簽約週期: 3-8 週

---

### 版本 1A：直接推薦式（最常用）

**Subject**: 科技人才推薦 - Java/Frontend/PM engineer｜現成候選人

---

嗨，

我是 Phoebe，台灣科技人才獵頭顧問。

看到你們最近在招 [職位名] ，我手上剛好有 2-3 位非常符合的候選人——都經過電話篩選，確認薪資期待、地點、入職時間都吻合。

**簡介一位，你感受一下：**

✦ **年資**: 9+ 年 [核心技能]  
✦ **現職**: Sr. Engineer @ [前公司]  
✦ **核心經歷**: [高流量系統 / 開源貢獻 / 關鍵產品]  
✦ **技術棧**: [Java / React / DevOps 等]  
✦ **市場價位**: 推估 [薪資範圍]  

**為什麼推薦給你們？**
- 技術棧完全吻合你們的需求
- 已離職 / 有 notice，可快速上線
- 對新創環境有興趣（不是被迫換工作）
- 我電話篩選過，確認他真的想要這份工作

**時間價值**: 
省掉你們 3-4 週的篩選時間。你們可以直接約面試，不用再篩 100 份履歷。

方便 15 分鐘電話嗎？我可以解答更多細節，確認契合度。

Cheers,  
Phoebe

## 💼 Tier 2：上市公司 開發信

### 特徵
- HR 流程複雜

我是 Phoebe，資深獵頭。

考慮和貴公司在 [職位] 招聘上有合作的可能性。我的服務模式是：替貴公司代為初篩候選人，確保送給你們的簡歷都經過電話驗證，不只是「履歷符合」，而是「真的適配」。

**我目前手上的候選人：**
- 9+ 年 [技能] — Sr. Engineer @[前公司]
- 年薪期待: [範圍]
- 可上線時間: [多少天]

**我的篩選流程：**
✓ 電話驗證技能 & 工作經驗  
✓ 確認薪資期待、地點、時間

✓ 只推薦「真的想要」的候選人

**對貴公司的好處：**
- 減少 HR 篩選履歷的時間（40-50%）
- 降低面試後 candidate decline 率
- 我負責初篩，你們直接面試

如果有興趣，我可以先發匿名簡介給你，確認符合需求再深入討論合作方式。

期待聽到你的想法。

Cheers,

## 🎯 開發信核心要素檢查表

### ✅ 必須有的：
- [ ] 稱呼具體（不是「Hi there」）
- [ ] 自我介紹簡短（1 句）
- [ ] 立即價值（「我有現成的 X」）
- [ ] 候選人亮點（3-4 點核心優勢）
- [ ] 推薦理由（為什麼適合他們）
- [ ] CTA 清晰（15 分電話，不是「有興趣嗎」）

### ❌ 絕不能有的：
- ❌ 長篇大論解釋你是誰
- ❌ 完整候選人簡歷（應該是「匿名簡介」）
- ❌ 公司背景介紹（他們會自己查）
- ❌ 售後承諾（「保證 X 天內...」）
- ❌ 過度親暱（除非真的認識）

## 💬 回應期望與話術

### 他們說「暫時沒有這職位」

```
回應: 「理解。不過我這邊會持續關注有沒有符合的機會，
      需要的時候直接聯絡你？」

目的: 保持在他們的「人才池」裡，未來有需求時直接接觸
```

### 他們說「你可以發匿名簡介」

```
立即行動: 24 小時內寄出
格式: 
- 名字: [匿名] 
- 年資: 9 年
- 核心經驗: [要點]
- 薪資期待: [範圍]
- 可上線: [時間]

郵件: 「已寄上，請看一下。符合的話，我們可以安排 30 分
      電話，我會讓他自我介紹，回答你們所有問題。」
```

### 他們說「費用太高」

```
回應: 「完全理解。我們費用是按市場行情，
      但如果你有特定預算限制，我可以看看能不能幫忙調整。」

轉換: 改為「月費制」或「固定費用」（降低他們 upfront 成本）
```

### 他們說「我們在和其他獵頭合作」

```
回應: 「沒問題。我這邊如果有特別適合的人選，
      還是會推薦給你們參考。」

目的: 保持開放，等他們對其他獵頭失望時再推進
```

## 📞 電話篩選腳本（15 分鐘，給雇主）

```
Opening (2 min):
「嗨 Kevin，謝謝你撥空聊天。
 我就直接進入重點——你們現在在招什麼職位？
 時間上多急？」

Discovery (5 min):
Q1: 「具體缺什麼職位？需要幾個 headcount？」
Q2: 「理想候選人是什麼樣的？年資、技能、薪資期待？」
Q3: 「有其他獵頭在幫你們找嗎？」
Q4: 「之前的招聘過程怎樣？痛點是什麼？」

Pitch (4 min):
「基於你說的需求，我正好有 2-3 位候選人滿符合。
 他們都是 [年資] 的 Sr. Engineer，技術棧跟你們一樣。
 我已經電話篩選過，確認薪資期待、入職時間都吻合。
 
 要不要我先寄匿名簡介給你看？
 如果符合，我們可以下周安排面試。」

Close (2 min):
「好，我週三把簡介寄給你。
 你看一下，覺得有興趣，我們再約面試時間？」
```
