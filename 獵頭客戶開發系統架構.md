# 獵頭客戶開發系統架構

**目標**: 建立系統性的客戶（雇主）開發流程  
**角色**: Phoebe 作為專業獵頭顧問  
**時間軸**: 3-6 個月內建立穩定客戶管道

---

## 📊 整體架構

```
客戶池定義
    ↓
客戶分層 (Tier 1/2/3)
    ↓
開發渠道 (多點觸擊)
    ↓
價值定位 (為什麼選我們)
    ↓
進度漏斗管理 (Pipeline)
    ↓
轉化與合作簽約
    ↓
長期維護 (客戶生命週期)
```

## 🎯 第一層：客戶池定義

### 誰是我們的「客戶」？

按 **公司規模** 分類：

| 階層 | 規模 | 特徵 | 優先度 |
|------|------|------|--------|
| **Tier 1** | 中型新創 (Series A/B) | 快速成長、缺人多、有預算、決策快 | ⭐⭐⭐ |
| **Tier 2** | 上市科技公司 | 穩定、招聘頻繁、流程慢但金額大 | ⭐⭐⭐ |
| **Tier 3** | 小新創/新團隊 | 缺人但預算小、高風險 | ⭐ |
| **Tier 0** | 國際科技公司 | 大單、複雜、長期 | ⭐⭐ (未來) |

### 我們優先開發的客戶特徵

**必須符合**:
- ✅ 科技公司（工程師為主）
- ✅ 有實際招聘需求（不是「可能」，而是「現在」）
- ✅ 有預算（不一定高，但要真實）
- ✅ 決策者可接觸（CEO / HR / Tech Lead）

**優先選擇**:
- 🟢 Java / DevOps / Frontend / PM / Data 領域（我們有候選人）
- 🟢 台北/新竹（地點相同，便於溝通）
- 🟢 融資後 6-12 個月內（成長最快、缺人最急）

## 🔗 第二層：客戶分層策略

### **Tier 1 新創：快速成長型（優先開發）**

**代表公司**: Playsee, AIJob, 宇泰華, CMoney 等

**特徵**:
- Series A/B 融資
- 月薪資 5-8 萬起
- 3-6 個月一個招聘週期
- CEO/CTO 通常親自參與招聘

**開發策略**:
```
接觸方式: 直接 LinkedIn DM / Email 給 CEO 或 CTO
頻率: 每月 2-3 次接觸
價值主張: 「我有現成的高品質候選人，
          可以節省你們 2-4 週的篩選時間」
佣金模式: 5-8% 或月費 5-10k
預期周期: 3-8 週內簽約
```

**開發信重點**:
- 直接說「我有候選人」，不是「我能幫你找」
- 具體數據（12+ 年經驗、XX 專技能）
- 強調時間成本節省
- 立即 CTA：「方便 15 分鐘電話嗎？」

---

### **Tier 2 上市公司：穩定流量型**

**代表公司**: Yahoo, 91APP, 玉山銀行, 富邦等

**特徵**:
- 每季度 5-10 個職缺
- 薪資 8-15 萬起
- HR 流程複雜但錢多
- 轉介費用高（10-15%）

**開發策略**:
```
接觸方式: Email 給 HR Manager / Recruiter
頻率: 每月 1 次定期推薦
價值主張: 「我提供預篩選候選人，
          降低你們 HR 篩選成本」
佣金模式: 8-12% 或按年合作關係
預期周期: 4-12 週（流程長）
```

**開發信重點**:
- 尊重他們的流程
- 強調「代為初篩」的專業性
- 與其 HR 建立長期關係
- 定期提供「人才市場情報」維繫關係

---

### **Tier 3 小新創：高風險但高回報**

**代表公司**: 未融資 / Pre-seed 新創

**特徵**:
- 缺人但預算小（月薪 3-5 萬）
- 決策快（CEO 自己搞）
- 容易失敗（倒閉的風險）

**開發策略**:
```
接觸方式: 朋友轉介 / 群組推薦
頻率: 機會性，不主動開發
價值主張: 「我們幫你找對人，
          減少早期團隊的找人成本」
佣金模式: 低於市場（5% 或固定 20-30k）
預期周期: 2-4 週（快速決策）
風險: 高（公司倒閉則費用白費）
```

## 📍 第三層：多渠道開發策略

### 渠道 1：直接主動開發（50% 時間投入）

**目標**: 每月主動聯繫 30-50 家公司

**流程**:
```
Step 1: 名單建立
- 用 LinkedIn 搜尋「Java Engineer」「Frontend」等相關職位
- 篩選「Series A/B 新創」或「科技公司」
- 找出 Hiring Manager / CEO / CTO

Step 2: 人物研究 (5 分鐘/人)
- LinkedIn 檔案：背景、公司情況
- 公司官網：融資信息、最近新聞
- 招聘頁面：目前招聘職位清單

Step 3: 個性化開發信
- 提及具體公司名和職位需求
- 說「我有 X 個現成候選人符合 Y 職位」
- 強調時間成本節省

Step 4: 多渠道觸及
- Day 1: LinkedIn DM
- Day 5: Email (if found)
- Day 10: LinkedIn Follow-up
```

**每月投入計算**:
- 30 家公司 × 20 分鐘 = 10 小時
- 預期回應率: 20-30%
- 預期簽約: 1-2 個客戶/月

---

### 渠道 2：轉介與關係（30% 時間投入）

**目標**: 透過現有客戶、朋友轉介新客戶

**方法**:
```
1. 現有客戶轉介
   - 「你認識其他在找人的公司嗎？」
   - 轉介成功給獎金 (1-2 萬)

2. 獵頭社群轉介
   - 加入台灣獵頭 / HR 社群
   - 分享 case、互相轉介

3. 天使投資人轉介
   - 接觸新創天使投資人
   - 他們掌握大量新創資訊

4. 顧問轉介
   - 與企管顧問、CFO 建立關係
   - 他們常知道客戶的招聘需求

5. 朋友與人脈
   - 「Warm introduction」轉介最有效
   - 回應率 60-80%
```

**預期效果**:
- 轉介客戶簽約率: 50-70%（質量高）
- 長期客戶關係: 更穩固

---

### 渠道 3：內容與品牌（20% 時間投入）

**目標**: 讓客戶主動找上門

**方法**:
```
1. LinkedIn 定期發文
   - 週 2-3 篇內容
   - 「前端工程師市場洞察」
   - 「新創招聘常見坑」
   - 「我幫企業找到的優秀人才故事」

2. 行業報告 / 白皮書
   - 每季發佈「科技人才市場報告」
   - 薪資範圍、稀缺技能、市場趨勢
   - 免費分享，建立權威性

3. Podcast / 直播
   - 邀請 CEO / CTO 討論「新創如何快速組建團隊」
   - 自然展示專業度

4. 社群活動參與
   - 新創活動、TC Pitch Day 等
   - 現場接觸 CEO 和 HR

5. 公開案例研究
   - 「我如何在 2 週內幫 XXX 新創找到 Sr. Engineer」
   - 具體數據、結果展示
```

**預期效果**:
- Inbound Lead 率: 每月 2-5 個
- 品牌信任度: 持續提升

## 🔄 第四層：進度漏斗管理（Pipeline）

### 客戶開發 Pipeline 分層

```
冷接觸池 (Cold Prospect)
    ↓ (20-30% 回應)
初步接觸 (Initial Contact)
    ↓ (50-60% 推進)
需求確認 (Needs Discovery)
    ↓ (70-80% 進入)
推薦候選人 (Candidate Submission)
    ↓ (40-60% 進展)
簽約合作 (Contract Signed)
    ↓ (80-90% 執行)
候選人入職 (Candidate Placement)
    ↓
客戶保留 (Customer Retention)
```

### 各階段目標與時間

| 階段 | 目標 | 時間 | 行動 | 成功率 |
|------|------|------|------|--------|
| **冷接觸** | 建立名單 | Day 0 | LinkedIn 搜索 | N/A |
| **初步接觸** | 獲得回應 | Day 1-3 | DM / Email | 20-30% |
| **需求確認** | 了解招聘需求 | Day 4-10 | 15 分電話 | 50-60% |
| **推薦候選人** | 提交合適人選 | Day 11-20 | 發送簡歷 | 70-80% |
| **簽約合作** | 確認合作協議 | Day 21-45 | 簽署合約 | 40-60% |
| **執行** | 候選人進度 | Day 46+ | 跟進面試 | 80-90% |

### 漏斗數據範例

```
100 個冷接觸
  ↓ (25% 回應)
25 個初步接觸
  ↓ (60% 推進)
15 個需求確認電話
  ↓ (80% 進入)
12 個推薦候選人
  ↓ (50% 簽約)
6 個簽約客戶
  ↓ (85% 成功放置)
5 個候選人入職
```

**月目標計算**:
- 要達成 5 個月度放置
- 需要 100 個冷接觸
- = 每天 3-5 個新客戶接觸

## 💼 第五層：價值定位（為什麼客戶選我們）

### 對不同客戶的價值主張

#### **給 Tier 1 新創**:
```
「我有現成的高品質候選人，
 不用你們自己招聘、篩選、面試，
 可以省 4-6 週時間，
 讓你們專注產品而不是招聘。」

ROI: 時間 >> 金錢
```

#### **給 Tier 2 上市公司**:
```
「我代為初篩候選人，
 確保送給你們的簡歷都經過電話驗證，
 不是履歷濾水，而是真的適配。
 減少你們 HR 的篩選成本 40-50%。」

ROI: 流程效率 & 成本降低
```

#### **給 HR / Recruiter**:
```
「我是你的外包篩選團隊，
 高峰期有職缺時，
 你可以直接用我們的候選人池。
 低峰期沒工作量，
 不佔你們正職成本。」

ROI: 彈性成本結構
```

## 📞 第六層：轉化策略

### 電話篩選腳本（15 分鐘）

```
Opening (2 min):
「嗨 Kevin，我是 Phoebe。
 謝謝你撥空聊天。
 我就直接切入重點——
 你們公司現在在找什麼職位？」

Discovery (7 min):
Q1: 「具體缺什麼職位？有幾個 headcount？」
Q2: 「理想候選人的年資、技能、期待薪資是多少？」
Q3: 「時間緊急嗎？什麼時候要上線？」
Q4: 「你們用什麼渠道找人？效果怎樣？」

Pitch (4 min):
「基於你說的需求，我手上剛好有 2-3 位候選人很符合，
 他們都經過電話篩選，確認薪資期待、地點、入職時間都吻合。
 如果你有興趣，我可以先寄匿名簡介給你看。」

Close (2 min):
「那我週三把簡介寄給你，
 你看過後，覺得符合再約候選人嗎？」
```

### 簽約談判

**佣金模式**（選一種）:
```
1. 按比例佣金 (最常見)
   - Tier 1 新創: 5-8% 年薪
   - Tier 2 上市: 10-15% 年薪
   - 計算: 年薪 100 萬 × 10% = 10 萬

2. 月費制 (適合大量招聘)
   - 月費: 5-20 萬
   - + 固定佣金: 2-3%
   - 適用: 持續招聘 5 人/月 以上

3. 固定費用 (小案子)
   - 單一職位: 20-40 萬
   - 適用: 緊急 1-2 人

4. 混合模式 (推薦)
   - 月費 10 萬 (baseline)
   - + 超額佣金 3-5% (鼓勵超額放置)
```

**簽約文件**:
- [ ] 服務協議（定義職位範圍、佣金、保證期）
- [ ] 候選人保密協議
- [ ] 付款條款（通常 candidate onboard 後 30 天內）

## 🎯 第七層：長期客戶維護

### 客戶生命週期管理

```
簽約後 (Onboarding)
    ↓ (維繫關係，防止流失)
定期推薦 (Regular Candidate Submission)
    ↓ (建立信任，增加金額)
擴展合作 (Account Expansion)
    ↓ (轉介新客戶)
品牌倡導 (Customer Advocacy)
```

### 維護策略

**每月溝通**:
- [ ] 1 次 15 分鐘進度電話（檢查招聘進度）
- [ ] 發送「人才市場情報」（展示價值）
- [ ] 推薦 1-2 個候選人（保持連結）

**季度檢查**:
- [ ] 檢視本季放置情況（KPI review）
- [ ] 討論下季招聘計畫（提前規劃）
- [ ] 是否有其他職位可代找（擴大金額）

**年度策略**:
- [ ] 簽署續約或調整條款
- [ ] 提供「年度人才市場報告」
- [ ] 轉介新客戶（建立推薦獎金）

### 防止流失

**警示信號**:
- ⚠️ 2 個月沒有新職缺
- ⚠️ HR 人員更換（需重新建立關係）
- ⚠️ 轉向其他獵頭（失去排他性）

**挽救行動**:
```
「嗨 Kevin，最近一陣子沒聽到你們的招聘消息，
 怎樣？產品發展順利嗎？
 我有幾個優秀候選人想介紹，
 不一定現在要，但先存著也不錯。」
```

## ✅ 核心要點

**客戶開發不是「廣告」，而是「關係建立」**:
1. 做功課（知道他們要什麼）
2. 提供價值（我有現成的高品質候選人）
3. 快速簽約（流程簡單，佣金合理）
4. 長期維護（定期接觸，防止流失）

**獵頭的真正競爭力**:
- 不是「我能幫你找人」（HR 自己會）
- 而是「我有現成的人才庫 + 篩選機制 + 節省你時間」
