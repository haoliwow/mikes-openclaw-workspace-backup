## 📞 面試提問清單（8 大問題）

### **Block 1：Spring Cloud & 微服務架構（核心技術）**

#### ❓ Q1：Spring Cloud 實戰經驗
```
「你的履歷提到了 SpringBoot，但沒有明確提到 Spring Cloud。
 能分享一下你在 Athena 或 Caresys 有沒有接觸過微服務架構？
 比如服務拆分、服務間通訊、負載均衡、熔斷器這些。」

評估指標：
- 有沒有用過 Spring Cloud 系列（Eureka/Feign/Hystrix/Config）
- 有沒有經歷過單體應用→微服務的轉型
- 理解微服務的核心 trade-off（性能 vs 複雜度）
```

#### ❓ Q2：分散式系統挑戰
```
「你提到做過資料庫遷移和零停機部署。
 在這個過程中，遇過什麼「分散式系統」的典型問題？
 比如資料一致性、分布式事務、還是網路分割？」

評估指標：
- 理解分散式系統 consistency vs availability 的 trade-off
- 有沒有實際解決過分散式鎖、兩段提交等問題
- 對系統可靠性的思考深度
```

---

### **Block 2：資料庫 & 性能優化（亮點能力）**

#### ❓ Q3：30x 性能提升案例細節
```
「履歷裡提到『30x 性能增長』在 database ingestion。
 具體是怎麼做的？用了什麼技術方案？」

評估指標：
- 是否真的理解 ThreadPoolExecutor 的配置（core/max threads, queue type）
- 有沒有考慮過背壓（backpressure）、批量處理、資料庫連線池
- 有沒有做過性能測試、監控、調優的完整流程
- 比例：是否是真實的 30x，還是特定場景
```

#### ❓ Q4：Oracle → PostgreSQL 遷移的架構挑戰
```
「Oracle 和 PostgreSQL 有明顯差異（SQL 方言、序列、資料型別）。
 你是怎麼用『資料庫抽象層』來統一支援兩個系統的？
 有沒有遇到不相容的情況？」

評估指標：
- 有沒有用過 JPA/Hibernate，理解 ORM 的優缺點
- 有沒有考慮過資料庫遷移時的向後相容性
- 對 PostgreSQL 特性的了解（JSON、UUID、JSONB 等）
- 架構抽象能力
```

### **Block 3：系統設計 & 決策能力**

#### ❓ Q5：Keycloak SSO 零停機遷移

```
「SSO 是系統的核心，遷移中斷會影響所有使用者。

- 風險評估與應急預案
```

#### ❓ Q6：Arthas 診斷工具的應用
```
「你用 Arthas 做實時診斷和修復。

評估指標：
- 理解 JVM 診斷工具（不只是會用，能解釋為什麼）
- 有沒有遇過記憶體洩漏、死鎖、高 CPU 等問題
- 故障處理的方法論（問題定位→分析→修復→驗證）
```
