# Step1ne API 候選人匯入格式指南

## 核心規則（Phoebe 2026-03-19 確立）

**黃金法則**：
- ✅ **無論如何都要匯入**（不能拒絕）
- **有開發信** → 狀態「已聯繫」
- **沒開發信** → 狀態「備選人才」

## 必填欄位（10 項）

| 欄位 | 類型 | 說明 | 範例 |
|------|------|------|------|
| `name` | string | 候選人姓名 | 「陳政瑋」或「Aiden Chen」 |
| `email` | string | 電郵 | aiden.chen@example.com |
| `phone` | string | 電話 | 0912-345-678 |
| `current_position` | string | 現職職位 | 「Senior Backend Engineer」 |
| `current_company` | string | 現職公司 | 「KIAO」 |
| `total_years` | string | 總年資（字串） | 「3.67」 |
| `skills` | string | 技能（逗號分隔） | 「Java, Spring Boot, PostgreSQL, Docker, K8s」 |
| `location` | string | 地點 | 「台北市」 |
| `source` | string | 來源 | 「CakeResume」或「LinkedIn」或「推薦」 |
| `recruiter` | string | 負責獵頭 | 「Phoebe」或「Jim」 |

---

## 推薦欄位

| 欄位 | 類型 | 說明 | 範例 |
|------|------|------|------|
| `linkedin_url` | string | LinkedIn 連結 | https://linkedin.com/in/... |
| `status` | string | 狀態 | 「已聯繫」或「備選人才」 |
| `stability_score` | number | 穩定度分數 (20-100) | 85 |
| `talent_level` | string | 綜合評級 | 「A」或「B」或「C」（只接受A/B/C，不接受A+） |
| `notes` | string | 備註 | 「職位：Java Backend Engineer #224」 |
| `education` | string | 教育背景（文字） | 「國立台灣大學 資訊工程系 學士 2020」 |

## 重要欄位：work_history（工作經歷）

**格式：** JSON 陣列（snake_case）  
**位置：** 同級欄位（與 name、email 同層）

### ✅ 正確格式

```json
{
  "name": "陳政瑋",
  "email": "aiden.chen@example.com",
  "current_position": "Backend Engineer",
  "total_years": "3.67",
  "skills": "Java, Spring Boot, PostgreSQL",
  "location": "台北市",
  "source": "CakeResume",
  "recruiter": "Phoebe",
  "status": "已聯繫",
  "work_history": [
    {
      "company": "KIAO",
      "title": "Backend Engineer",
      "start": "2023-01",
      "end": "present",
      "description": "開發支付系統、實現 gRPC 微服務架構"
    },
    {
      "company": "TechCorp",
      "title": "Junior Backend Engineer",
      "start": "2021-06",
      "end": "2022-12",
      "description": "負責 API 開發與資料庫最佳化"
    },
    {
      "company": "Startup XYZ",
      "title": "Full Stack Developer",
      "start": "2020-08",
      "end": "2021-05",
      "description": "全棧開發、前後端整合"
    }
  ]
}
```

### ❌ 常見錯誤

- ❌ `work_history` 為物件而非陣列（應該是 `[{...}, {...}]`）
- ❌ `start`/`end` 用完整時間戳而非 YYYY-MM 格式
- ❌ `title` 拼成 `position` 或 `job_title`
- ❌ `description` 為空或過短（應該有實際職責描述）

## education_details（教育背景）

**狀況**（2026-03-05 Phoebe 確認）：
- API 接受此欄位，但**不會持久化到資料庫**
- UI 顯示會是空白（後端邏輯問題）
- **替代方案**：使用 `education` 欄位（文字格式）

### 使用 education 欄位（推薦）

```json
{
  "education": "國立台灣大學 資訊工程系 學士 2020 | 某補習班 AWS 認證 2021"
}
```

## 檢查清單

每位候選人匯入後驗證：
- [ ] 建檔成功（ID 已生成）
- [ ] Email 有效（不重複）
- [ ] `total_years` 使用字串格式（不是數字）
- [ ] `work_history` 是 JSON 陣列且完整
- [ ] `talent_level` 只用 A/B/C（不用 A+）
- [ ] `status` 為「已聯繫」（有開發信）或「備選人才」（無開發信）
- [ ] 職位資訊編碼在 `notes` 欄（因為 API 無法直接更新 job_position）

## 常見錯誤與修正

| 錯誤 | 修正 |
|------|------|
| `total_years` 為數字（3.67） | 改為字串（"3.67"） |
| `talent_level` 為 "A+" | 改為 "A"（系統只接受 A/B/C） |
| `work_history` 為物件 `{...}` | 改為陣列 `[{...}, {...}]` |
| `job_position` 欄位無法更新 | 用 `notes` 欄位代替（職位：#224） |
| `education_details` 不顯示 | 用 `education` 欄位替代 |
